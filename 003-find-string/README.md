# 2026.3.28 找出对应`LCP`矩阵的字符串

## 题目描述

对任一由 `n` 个小写英文字母组成的字符串 `word` ，我们可以定义一个 `n x n` 的矩阵，并满足：

- `lcp[i][j]` 等于子字符串 `word[i,...,n-1]` 和 `word[j,...,n-1]` 之间的最长公共前缀的长度。

给你一个 `n x n` 的矩阵 `lcp` 。返回与 `lcp` 对应的、按字典序最小的字符串 `word` 。如果不存在这样的字符串，则返回空字符串。

对于长度相同的两个字符串 `a` 和 `b` ，如果在 `a` 和 `b` 不同的第一个位置，字符串 `a` 的字母在字母表中出现的顺序先于 `b` 中的对应字母，则认为字符串 `a` 按字典序比字符串 `b` 小。例如，`"aabd"` 在字典上小于 `"aaca"` ，因为二者不同的第一位置是第三个字母，而 `'b'` 先于 `'c'` 出现。

## 解决方法

个人：先通过给出的`lcp`矩阵构造出字符串，再通过字符串生成新的`lcp`矩阵，通过判断两个矩阵是否相同来决定是否返回字符串，但是由于**时间复杂度**过大无法通过测试

研究答案解法后，在个人的解法上进行了优化，发现原解法在`findLcp`函数部分用来三层循环，时间复杂度为O(n<sup>3</sup>)，无法合理解决n  <= 1e<sup>3</sup>的情况。于是发现答案在生成新的`lcp`矩阵这一步，只用了两层循环，于是我模仿这一步，从用于计算的字符串列表中最后一位往前生成新的l`cp`， 而不是从前往后生成，少了一个循环。这是用来**动态规划**的思想，利用子问题的解，解决大问题，由于后面的`lcp`已经算完了（最后一位天然为1），不断往前就可以得出结果。
